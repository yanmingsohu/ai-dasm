# AI-Based Disassembly Engineering

## Disassembly

```sh
python dasm.py \
  -f [EXE file path] \
  -o [output directory]
```

This command reads the EXE and generates .c / .asm files in the output directory,
and generates a .callgraph.json function call graph file in the same directory as the EXE.


## Analysis

```sh
python logical_analysis.py \
  -a [usually the output directory of dasm.py, used to read .c/.asm files] \
  -o [should be the same as -a, outputs an analysis report .md] \
  -g [.callgraph.json file path] \
  -sys [system prompt file] \
  -api [OpenAI API-like endpoint URL] \
  -key [API key] \
  -model [AI model name]
```

This command uses a large language model to analyze the generated function files 
and produce a report.


## Generate Cutter Comments

```sh
python gen_rizin.py \
  -m [directory of reports generated by logical_analysis.py] \
  -a [output directory of dasm.py; if not provided, the path from -m will be used] \
  -g [path to the .callgraph.json file] \
  -o [output path for the generated rizin script; if not provided, the path from -g will be used]
```

This command generates a rizin script. Executing this script in Cutter will merge the analysis report into Cutter as comments.


## Generate C Code

```sh
python source.py \
  -a [usually the output directory of dasm.py, used to read .asm files] \
  -o [directory to generate .c/.o files] \
  -sys [system prompt file] \
  -api [OpenAI API-like endpoint URL] \
  -key [API key] \
  -model [AI model name]
```

This command reads .asm files, then uses AI to generate corresponding C source files,
and compiles them with gcc to produce .o files.

Only compilation success is guaranteed; the logic may be incorrect.
